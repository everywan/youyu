# 财务自由辅助 App 产品定义

> 状态：当前产品定义与 MVP 实现基准；MVP 内容作为本文最后一个独立大章节维护。

## 一句话定位

这是一个本地隐私优先的个人财务自由规划仪表盘，用低频月度输入记录资产、完整生活预算和非资产收入，持续判断自己离基础 / 标准 / 高级财务自由还有多远。

核心差异点：

> 预算驱动的财务自由推演，而不是流水驱动的记账。

它不回答“我每一笔钱花到哪里了”，而是回答：

1. 我现在离财务自由还有多远。
2. 我的资产收益覆盖了多少预算支出。
3. 如果我多存钱、少花钱、提高收益率，会提前多久。
4. 如果预留一部分资产给房子、家庭、应急或下一代，我还剩多少自由空间。

## 目标用户

第一版先为个人使用设计。

适合的人：

1. 有财务自由目标，希望获得长期选择权。
2. 有储蓄和投资资产，但不想每天记账。
3. 愿意每月维护一次资产、收入和生活预算。
4. 关心资产收益能否覆盖生活成本。
5. 重视隐私，希望财务数据优先保存在本地。

不优先服务的人：

1. 只想快速记一笔消费的人。
2. 需要投资推荐、买卖信号或资产调仓建议的人。
3. 需要自动同步微信、支付宝、银行卡流水的人。
4. 需要多人家庭协作、云同步、完整账单导出的人。

## 使用场景

### 首次设置

用户第一次使用时，需要完成“用哪三档预算定义自由”的设置：

1. 录入基础 / 舒适 / 理想三档预算。
2. 录入当前收益型资产、非收益型资产、预留资产和不可动用资产。
3. 录入工资、公积金和其他非资产收入。
4. 系统生成初始财务自由进度。

### 每月更新

用户每月维护一次财务状态，而不是每天逐笔记账：

1. 更新收益型资产和非收益型资产金额。
2. 更新本月工资、公积金、其他非资产收入和预算后结余。
3. 更新预算执行情况。
4. 查看自由等级、覆盖率、月净收入、消费对应的劳动时间和预计达成时间变化。

### 决策推演

用户在做收入、支出、资产预留或生活方式决策时，通过推演理解影响：

1. 每月多存一笔，会提前多久。
2. 支出提高到舒适预算，会推迟多久。
3. 预留买房、家庭、应急资金后，自由空间还剩多少。
4. 收益率变化会如何影响长期达成时间。

## 产品边界

### 做什么

1. 财务自由目标：基础 / 标准 / 高级三档由基础 / 舒适 / 理想预算定义。
2. 收益型资产记录：记录现金、存款、基金、股票、债券、黄金、房产等资产。
3. 非收益资产记录：记录房产等不产生收益的资产，金额完全由用户填写，不推导估值或贷款。
4. 预算管理：支持基础预算、舒适预算、理想预算三档；房贷月供、房租、保险等全部持续支出都进入预算。
5. 月度收入：记录工资、公积金和其他非资产收入，不做逐笔账本。
6. 财务自由仪表盘：展示自由等级、覆盖率、收益型资产、年资产收益、月净收入、消费对应的劳动时间和预计达成时间。
7. 推演：调整收入、支出、收益率、预留资产，实时查看自由时间变化。
8. 本地数据保存：默认本地优先，保证可长期自用。

### 不做什么

1. 不做传统记账：不要求逐笔消费录入。
2. 不做账单导入和账单导出。
3. 不做自动同步微信、支付宝、银行卡。
4. 不做复杂消费分类、商户识别或流水清洗。
5. 不做投资推荐、交易、调仓或收益承诺。
6. 不依赖股票、基金、黄金每日净值自动更新。
7. 不做多用户协作和云同步。

### 后续可做

1. CSV / Excel 账单导入，用于校准预算，而不是转成记账主线。
2. 理财产品价格更新，用于减少手动录入资产净值。
3. 月度财务报告，解释本月自由进度变化。
4. AI 总结和建议，但必须基于用户数据，不替代投资建议。
5. 数据备份恢复、JSON / CSV 导入导出。

## 核心概念

### 收益型资产

能够产生收益、现金流或长期增值可能的资产。

示例：

1. 现金、活期、定期存款。
2. 基金、股票、债券、黄金、理财。
3. 可出租房产、商铺等租金资产。

产品处理：

1. 进入资产记录。
2. 参与收益型资产、可支配资产和资产收益计算。
3. 可标记为可支配、不可动用或预留资产。

### 支出与贷款

产品不单独维护负债余额或还款计划。房贷月供、房租、保险等持续支出与其他生活成本一样，统一放入三档预算。用户每月根据当前情况自行调整预算。

房产等非收益资产的金额由用户直接输入；产品不要求录入可变现价值、剩余贷款，也不自动计算房产净值。

### 预算

预算是财务自由门槛，不只是支出限制。

产品支持三档预算：

1. 基础预算：最低可接受生活成本，对应基础自由。
2. 舒适预算：稳定、体面、有余地的生活成本，对应标准自由。
3. 理想预算：更自由、更高质量的生活成本，对应高级自由。

三档预算就是三档自由度的定义。系统判断某一档自由，不看主观标签，而看所有收益型资产按当前整体年化产生的资产收益，是否能覆盖该档预算对应的月 / 年生活成本。

### 耐用消费成本

手机、电脑、车、家电等不是财务自由模型里的资产，而是生活成本的一类。

它的原则是：

1. 购买时是一次性支出。
2. 使用时可按使用周期摊销为生活成本。
3. 二手售出金额是残值，用于降低本轮使用成本。
4. 它不进入收益型资产，也不参与资产收益计算。

完整功能设计见 `产品文档/耐用消费成本功能spec.md`。MVP 第一阶段只保留简化成本口径，避免拖慢核心闭环。

### 可支配资产

可支配资产用于判断“真正能支撑自由生活的钱”。

```text
可支配资产 = 收益型资产总额 - 不可动用资产 - 预留资产
```

不可动用资产示例：

1. 自住房中不打算出售的部分。
2. 给家庭、父母、孩子预留的资产。
3. 短期不能动用或动用成本很高的资金。

## 财务自由判断原则

产品默认用年口径预算覆盖判断自由等级：

| 等级 | 判断 |
| --- | --- |
| 未自由 | 年资产收益 < 基础年预算 |
| 基础自由 | 年资产收益 >= 基础年预算 |
| 标准自由 | 年资产收益 >= 舒适年预算 |
| 高级自由 | 年资产收益 >= 理想年预算 |

旧版自定义目标只保留数据兼容，不进入当前 MVP 的自由判断。当前自由等级只看年资产收益是否覆盖对应年预算。

## 关键公式

### 收益型资产

```text
收益型资产总额 = 所有收益型资产金额合计
收益型资产 = 收益型资产总额
```

收益型资产总额用于计算资产收益。非收益资产和手机、电脑、车等耐用品不进入收益型资产；其金额只按用户输入展示，不能让财务自由目标的资产口径虚高。

### 可支配资产

```text
可支配资产 = 收益型资产总额 - 不可动用资产 - 预留资产
```

可支配资产用于判断扣掉不可动用资产和明确预留后，真正能支撑自由生活的钱。

### 月结余

```text
月收入 = 工资及公积金 + 其他非资产收入
月支出 = 固定支出 + 日常支出 + 家庭支出 + 年度支出本月折算 + 耐用消费本月摊销
月结余 = 月收入 - 月支出
```

### 月净收入与消费劳动时间

首页用月净收入把消费金额换算成真实劳动时间，帮助用户在大额消费前快速判断“这件东西值得我工作多久”，形成消费警戒。

```text
月资产收益 = 年资产收益 / 12
基础月预算支出 = 基础年预算支出 / 12
月净收入 = 当月持续现金流收入 + 月资产收益 - 基础月预算支出
赚 100 元需要的小时数 = 100 / (月净收入 / 160)
```

口径：

1. “现金流”取当前月持续性现金流收入；工资收入包含税后工资及个人、单位双方公积金，其他持续现金流收入也计入。资产收益按资产金额与年化收益率单独计算。
2. 第一版按每月 `160` 个工作小时估算，暂不要求用户维护工时。
3. 月净收入小于等于 `0` 时，不计算劳动时间，首页显示“无法覆盖”。
4. 该指标用于消费决策提醒，不代表实际时薪或劳动法意义上的小时工资。

### 年预算支出

```text
年预算支出 = 月预算 * 12 + 年度大额支出 + 年度预留支出
```

### 年资产收益覆盖率

```text
年资产收益 = Σ(收益型资产金额 * 该资产年化收益率)
年资产收益覆盖率 = 年资产收益 / 年预算支出
```

### 支撑年限

```text
年资金缺口 = max(年预算支出 - 年资产收益, 0)
支撑年限 = 可支配资产 / 年资金缺口
```

边界：

1. 年资金缺口为 0 时，支撑年限显示为“已覆盖”。
2. 可支配资产小于 0 时，提示不可动用或预留资产过高。
3. 月结余 <= 0 时，不输出乐观达成时间，只提示当前现金流无法自然达成目标。

### 预计自由时间

按月迭代：

```text
月收益率 = (1 + 所有收益型资产整体实际年化) ^ (1 / 12) - 1
本月资产收入 = 当前收益型资产总额 * 月收益率
下月收益型资产总额 = 当前收益型资产总额 + 非资产收入 + 本月资产收入 - 对应档位月预算
```

直到满足对应自由等级：

```text
年资产收益 = 收益型资产总额 * 所有收益型资产整体实际年化
年资产收益 >= 对应等级年预算支出
```

## MVP 方案

MVP 方案是本文的实现章节，用来说明第一版具体怎么做。它不改变前文产品定义，只把长期产品定义收敛成第一阶段可开发、可验证的闭环。

### MVP 目标

MVP 只验证一个核心闭环：

> 用户每月维护一次财务状态，系统能算出自由等级、年资产收益覆盖率、支撑年限、预计达成时间，并说明主要瓶颈。

MVP 成功标准：

1. 用户能录入三档完整生活预算、收益型与非收益型资产和月度非资产收入。
2. 系统能计算收益型资产总额、可支配资产、年预算支出、年资产收益、年资产收益覆盖率、资金缺口和支撑年限。
3. 系统能按基础 / 标准 / 高级三档预算判断当前自由等级。
4. 系统能通过推演展示收入、支出、收益率、预留资产变化对自由时间的影响。
5. 全流程不依赖逐笔记账、账单导入、联网行情或投资建议。

### MVP 范围

#### 必做

1. 预算设置：基础 / 舒适 / 理想三档预算，用于定义三档自由门槛。
2. 首页仪表盘：自由等级、覆盖率、收益型资产、年资产收益、月净收入、赚取 `100` 元所需劳动时间、预计达成时间和关键提醒。
3. 资产：收益型资产、非收益型资产、不可动用资产、预留资产、年资产收益。
4. 预算：基础 / 舒适 / 理想三档完整生活预算，包含房贷月供、房租、保险等全部持续支出。
5. 月度收入：工资、公积金、其他非资产收入和备注。
6. 推演：调整收入、支出、收益率、预留资产、不可动用资产，查看自由时间变化。
7. 本地保存：本地持久化和备份 / 恢复入口。

#### 简化处理

1. 耐用消费成本：MVP 第一阶段不做完整耐用品台账，只允许在月度现金流中录入“耐用消费本月摊销”或在预算中录入月均成本。
2. 退休预估：作为独立 spec 保留在 `产品文档/退休预估功能spec.md`，不阻塞 MVP 核心闭环。

#### 不做

1. 逐笔记账、商户识别、复杂分类。
2. 微信 / 支付宝 / 银行卡自动同步。
3. 账单导入、账单导出。
4. 投资推荐、交易、调仓或收益承诺。
5. 股票、基金、黄金每日净值自动更新。
6. 多用户协作、云同步。

### MVP 页面

#### 0. 预算设置

预算设置用于定义“什么叫自由”，首次使用必须完成。

MVP 支持：

1. 基础预算：基础自由的月支出门槛。
2. 舒适预算：标准自由的月支出门槛。
3. 理想预算：高级自由的月支出门槛。

默认判断顺序：

1. 用预算覆盖模型判断自由等级。
2. 首页展示每档资产收益达到多少/月，以及对应预算门槛多少/月。

#### 1. 首页：财务自由仪表盘

首页是产品核心，不是账本入口。

核心展示：

1. 当前自由等级：未自由 / 基础自由 / 标准自由 / 高级自由。
2. 年资产收益覆盖率。
3. 收益型资产。
4. 年资产收益。
5. 月净收入。
6. 赚取 `100` 元所需劳动时间，用于把大额消费换算成真实劳动时间。
7. 距离基础 / 标准 / 高级自由预计还需多久。
8. 关键提醒：预算偏高、结余不足、预留资产过高、资产收益不足。

首页四个摘要指标按两行展示：第一行为“收益型资产、年资产收益”，第二行为“月净收入、赚 100 元需要 x 小时”。

三档自由进度必须同时展示该档覆盖率和月口径解释：当前年资产收益折算到每月是多少、该档预算门槛每月是多少。例如“资产收益 5000/月 / 门槛 8000/月”，让用户知道差距来自哪一档预算。

#### 2. 资产页

资产页不做交易软件，只回答资产是否可支配、是否产生现金流、是否支持自由目标。

收益型资产字段：

1. 名称。
2. 类型：现金、存款、基金、股票、债券、黄金、房产、其他。
3. 当前金额。
4. 是否可支配。
5. 是否不可动用。
6. 是否预留资产。
7. 预留用途。
8. 预期年化收益率。
9. 年资产收益。
10. 更新时间。

非收益资产字段：

1. 名称。
2. 类型：房产、车辆、其他。
3. 用户填写的金额。
4. 更新时间。

非收益资产不参与资产收益和自由等级计算。产品不推导其可变现价值、剩余贷款或净值。

#### 3. 预算页

预算决定财务自由门槛。

MVP 支持：

1. 三档预算：基础、舒适、理想。
2. 月预算和年度大额支出。
3. 分类预算。
4. 年预算自动计算。
5. 不同预算档位对应的自由等级和预计自由时间。

#### 4. 收入页

MVP 不做逐笔记账，只做月度汇总。

字段：

1. 起始月份。
2. 工资。
3. 公积金比例。
4. 其他非资产收入。
5. 一次性收入。
6. 备注。

资产产生的收益由资产金额和年化收益率计算，不在收入页重复录入。所有支出统一进入预算。

工资现金流口径：

1. 月主动收入包含税后工资及个人、单位双方公积金，月结余、首页月净收入和预算推演均使用这一完整口径。
2. 资产实际入账时，税后工资进入现金余额，个人、单位双方公积金进入锁定的公积金余额。
3. 年终奖的金额、浮动系数以及随之发生的额外支出不稳定，默认不进入持续现金流和预算推演；实际到账后与意外收入、意外支出一样，作为一次性现金流单独录入和计算。

#### 5. 推演页

推演页用于回答“调什么变量能更快自由”。

推演结果随参数实时变化，不提供“保存为场景”或历史场景列表；推演参数只服务当前决策，不进入长期数据维护。

用户可调整：

1. 每月收入，工资场景包含税后工资及个人、单位双方公积金，不包含年终奖。
2. 基础 / 舒适 / 理想预算，即基础 / 标准 / 高级三档自由度门槛。
3. 预留资产。
4. 不可动用资产。

系统展示：

1. 基础自由时间。
2. 标准自由时间。
3. 高级自由时间。
4. 当前可支配资产支撑年限。
5. 每月非资产收入、资产收入和预算支出如何共同影响收益型资产总额。
6. 与当前状态相比提前或推迟多久。

推演过程集中展示以下口径说明：

1. 计算方式：工资及其他非资产收入 + 资产收益 − 对应档位完整预算。
2. 达成目标：当“（综合年化收益率 − CPI）× 总资产 ≥ 当前档年预算”时，视为达成。
3. 备注：工资收入包含税后工资及个人、单位双方公积金；年终奖、奖金、意外收入及意外支出等不确定因素默认不计入推演。综合年化收益率按当前资产配置加权计算。

### 验收场景

#### 场景一：小县城基础自由

输入：

1. 基础预算 `5000 / 月`。
2. 当前收益型资产总额、月结余、年资产收益。

系统应输出：

1. 年资产收益覆盖率。
2. 当前资产收益达到多少/月，基础自由门槛是多少/月。
3. 按当前月结余和收益率预计需要多久覆盖基础预算。

#### 场景二：城市打工人

输入：

1. 有房贷或房租。
2. 有每月主动收入和支出。
3. 有预留资产和应急金目标。

系统应输出：

1. 收益型资产。
2. 可支配资产。
3. 年资金缺口。
4. 应急金是否不足。
5. 当前状态下的基础自由时间。

#### 场景三：推演

输入：

1. 每月多存 `3000`。
2. 年化收益率从 `4%` 调整为 `5%`。
3. 预留资产增加 `300000`。

系统应输出：

1. 基础自由时间提前或推迟多少。
2. 标准自由时间提前或推迟多少。
3. 关键瓶颈是结余、收益率、预算还是预留资产。

#### 场景四：简化耐用消费成本

输入：

1. 本月耐用消费摊销 `400`。

系统应输出：

1. 该成本进入月度生活成本和预算。
2. 该成本不进入资产。
