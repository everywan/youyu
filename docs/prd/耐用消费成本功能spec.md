# 耐用消费成本功能 Spec

> 来源：从 `历史/MVP产品定义v1.md` 中拆出并收敛。
> 状态：后续功能；不阻塞 MVP 第一阶段核心闭环。
> 关联：`../产品定义.md`。

## 1. 功能定位

耐用消费成本用于记录手机、电脑、车、家电等一次性耐用品投入的长期生活成本。

它不把耐用品当作财务自由模型里的资产，而是把它们按使用周期摊销为生活成本，避免净资产虚高。

## 2. 与 MVP 的关系

MVP 第一阶段只保留简化口径：

1. 在月度现金流中填写“耐用消费本月摊销”。
2. 或在预算中填写一个月均耐用消费成本。
3. 不要求建立完整耐用品台账。

完整功能进入后，再支持购买记录、残值估算、售出校准和历史摊销。

## 3. 做什么

1. 记录耐用品名称、类型、购买金额、购买日期。
2. 记录预计使用月数和预计二手残值。
3. 自动计算预计净使用成本和预计月均使用成本。
4. 记录实际售出金额、实际售出日期和状态。
5. 售出后用实际金额校准历史月均使用成本。
6. 将月均使用成本进入预算和月度现金流分析。

## 4. 不做什么

1. 不把耐用品计入资产。
2. 不把残值计入净资产。
3. 不参与被动收入计算。
4. 不做二手价格自动估算。
5. 不做复杂折旧会计模型。

## 5. 数据模型

```ts
type DurableCost = {
  id: string
  name: string
  type: 'phone' | 'computer' | 'appliance' | 'car' | 'other'
  purchaseAmount: number
  purchaseDate: string
  expectedUseMonths: number
  expectedResaleValue: number
  actualSaleAmount?: number
  actualSaleDate?: string
  status: 'active' | 'sold' | 'discarded'
}
```

## 6. 计算公式

预计使用期：

```text
预计净使用成本 = 购买金额 - 预计二手残值
预计月均使用成本 = 预计净使用成本 / 预计使用月数
```

售出后校准：

```text
实际净使用成本 = 购买金额 - 实际售出金额
实际使用月数 = 实际售出日期 - 购买日期，按月折算
实际月均使用成本 = 实际净使用成本 / 实际使用月数
```

约束：

1. 预计二手残值不能高于购买金额。
2. 预计使用月数必须大于 0。
3. 未填写预计二手残值时按 0 处理。
4. 已售出物品使用实际售出金额校准。
5. 已报废物品实际售出金额按 0 处理。

## 7. 页面设计

耐用消费成本页展示：

1. 使用中物品列表。
2. 已售出 / 已报废物品列表。
3. 本月耐用消费摊销总额。
4. 每件物品的月均使用成本。
5. 购买金额、预计残值、实际残值的差异。

录入字段：

1. 名称。
2. 类型：手机、电脑、家电、车、其他。
3. 购买金额。
4. 购买日期。
5. 预计使用月数。
6. 预计二手残值。
7. 实际售出金额。
8. 实际售出日期。
9. 当前状态：使用中 / 已售出 / 已报废。

## 8. 验收场景

### 场景一：使用中物品

输入：

1. 电脑 `12000`。
2. 预计使用 `48` 个月。
3. 预计残值 `2000`。

期望：

1. 预计净使用成本为 `10000`。
2. 预计月均使用成本为 `208.33`。
3. 成本进入月度生活成本。
4. 物品不进入资产。

### 场景二：售出后校准

输入：

1. 电脑购买价 `12000`。
2. 实际使用 `36` 个月。
3. 实际售出 `3000`。

期望：

1. 实际净使用成本为 `9000`。
2. 实际月均使用成本为 `250`。
3. 后续使用实际月均成本校准历史统计。

### 场景三：报废

输入：

1. 手机购买价 `6000`。
2. 实际使用 `36` 个月。
3. 状态为已报废。

期望：

1. 实际残值按 `0` 处理。
2. 实际净使用成本为 `6000`。
3. 实际月均使用成本为 `166.67`。
