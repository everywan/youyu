# 财务自由辅助 App 退休预估功能 Spec

> 来源：2026-06-22 产品讨论。
> 状态：已确认设计，待用户复核；作为独立功能 spec 保留，并在 `2-build/projects.md` 中做摘要级登记。
> 关联：`MVP产品定义v1.md`。

## 1. 功能定位

退休预估是财务自由推演的一部分，不是社保计算器，也不是养老金精算工具。

它回答两个问题：

1. 退休前，靠当前资产增长、月结余和被动收入，能不能实现财务目标。
2. 如果退休前不能实现，退休后加上退休金，能不能覆盖目标预算。

退休预估采用“双阶段自由时间线”：

1. 退休前阶段：只使用现有财务自由模型。
2. 退休后阶段：在现有被动收入基础上加入退休金。

## 2. MVP 范围

### 做什么

1. 支持填写性别。
2. 支持填写当前年龄。
3. 根据性别自动使用默认退休年龄：男 63 岁，女 55 岁。
4. 支持手动填写预计退休后每月领取金额。
5. 在推演页展示退休前 / 退休后双阶段结果。
6. 在首页展示退休预估摘要。

### 不做什么

1. 不估算社保缴费政策。
2. 不根据是否继续缴纳社保自动调整退休金。
3. 不处理延迟退休政策变化。
4. 不做寿命预测。
5. 不做养老医疗支出预测。
6. 不做通胀模型。
7. 不把退休金算成资产；退休金只作为退休后现金流。

## 3. 输入与数据模型

### RetirementProfile

```ts
type RetirementProfile = {
  gender: 'male' | 'female'
  currentAge: number
  retirementAge: 63 | 55
  monthlyPensionIncome: number
}
```

字段说明：

1. `gender`：用户性别，只用于确定默认退休年龄。
2. `currentAge`：当前年龄。
3. `retirementAge`：系统根据性别自动设置，男 63，女 55。
4. `monthlyPensionIncome`：预计退休后每月领取金额，由用户手动输入。

规则：

1. `gender = male` 时，`retirementAge = 63`。
2. `gender = female` 时，`retirementAge = 55`。
3. MVP 不允许手动覆盖退休年龄。
4. MVP 只保留一个退休金金额，不区分“持续缴社保”和“财务自由后停缴社保”。

## 4. 计算逻辑

### 阶段一：退休前

从当前月份开始，按现有财务自由推演逻辑逐月计算：

```text
下月收益型净资产 = 当前收益型净资产 * (1 + 月收益率) + 月结余
下月可支配资产 = 下月收益型净资产 - 不可动用资产 - 预留资产
```

每个月判断是否达到预算覆盖目标：

```text
年被动现金流 >= 对应等级年预算支出
```

同时展示自定义辅助目标是否达成：

```text
收益型净资产 >= 目标资产
年被动现金流 >= 目标月被动收入 * 12
```

如果在退休年龄前达标，输出：

```text
退休前可实现财务目标
预计达成年龄：X 岁
预计达成时间：Y 年后
```

### 阶段二：退休后

如果退休前未达标，则从退休年龄开始加入退休金：

```text
年退休金 = 月退休金 * 12
退休后年收入覆盖 = 年被动现金流 + 年退休金
```

判断退休后是否覆盖目标预算：

```text
退休后年收入覆盖 >= 对应等级年预算支出
```

如果覆盖，输出：

```text
退休后可覆盖目标预算
退休后覆盖率：X%
退休后年度缺口：0
退休后支撑年限：已覆盖
```

如果仍不能覆盖，输出：

```text
退休后仍未覆盖目标预算
退休后年度缺口 = 对应等级年预算支出 - 退休后年收入覆盖
退休后支撑年限 = 可支配资产 / 退休后年度缺口
```

如果退休前已经达标，退休后仍展示退休后覆盖率，但不改变“达成时间”的结论。

## 5. 页面设计

### 目标设置

新增退休预估输入区：

1. 性别：男 / 女。
2. 当前年龄。
3. 默认退休年龄：男 63，女 55，只读展示。
4. 预计退休后每月领取金额。

如果未填写退休金，系统视为退休预估未启用。

### 推演页

新增“双阶段时间线”区域。

退休前阶段展示：

1. 是否能在退休前达成目标。
2. 预计达成年龄。
3. 距离退休还剩几年。
4. 若未达成，退休前缺口是多少。

退休后阶段展示：

1. 年被动现金流。
2. 年退休金。
3. 被动收入 + 退休金覆盖率。
4. 退休后年度缺口。
5. 退休后支撑年限。

### 首页仪表盘

首页只展示轻量摘要：

```text
退休前：预计无法达成 / 预计 51 岁达成
退休后：被动收入 + 退休金覆盖基础预算 86%
```

如果退休金未填写：

```text
退休预估未启用
```

## 6. 异常与边界

1. 未填写退休金：不展示退休后覆盖结果，只显示“退休预估未启用”。
2. 当前年龄已经达到或超过退休年龄：跳过退休前阶段，直接进入退休后阶段。
3. 月退休金为 0：退休后阶段仍可展示，退休金贡献为 0。
4. 退休后年度缺口 <= 0：支撑年限显示“已覆盖”。
5. 可支配资产 <= 0 且退休后仍有缺口：提示“退休后现金流仍不足，且可支配资产不足”。
6. 退休前已经达成目标：达成时间保持退休前达成时间，退休后覆盖率只作为补充信息。

## 7. 验收场景

### 场景一：退休前未达标，退休后达标

输入：

1. 男，当前 35 岁，退休年龄 63 岁。
2. 月退休金 4000。
3. 退休前未达到基础自由。
4. 退休后 `年被动现金流 + 年退休金 >= 基础年预算`。

期望：

1. 退休前阶段显示未达成。
2. 退休后阶段显示可覆盖基础预算。
3. 展示退休后覆盖率和年度缺口 0。

### 场景二：退休前已达标

输入：

1. 男，当前 35 岁，退休年龄 63 岁。
2. 预计 50 岁达到基础自由。
3. 月退休金已填写。

期望：

1. 系统显示预计 50 岁达成。
2. 退休后仍展示覆盖率。
3. 退休金不改变达成时间。

### 场景三：退休金未填写

输入：

1. 女，当前 45 岁，退休年龄 55 岁。
2. 退休金未填写。

期望：

1. 系统只展示退休前推演。
2. 退休后区域显示“退休预估未启用”。

### 场景四：当前已退休

输入：

1. 当前年龄 65 岁。
2. 退休金已填写。

期望：

1. 系统跳过退休前阶段。
2. 直接计算退休后覆盖率、年度缺口和支撑年限。

## 8. 与 MVP 产品定义的关系

退休预估不改变 MVP 的核心定位。

它仍然服务于：

1. 预算驱动的财务自由推演。
2. 低频月度输入。
3. 被动收入覆盖预算。
4. 可支配资产支撑年限。

它新增的是退休后的现金流补充判断：

```text
退休后收入覆盖 = 年被动现金流 + 年退休金
```

该功能不应引入账单、社保估算、投资推荐或复杂政策计算。
